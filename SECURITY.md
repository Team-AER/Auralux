# Security Policy

## Supported versions

This project is in active development. Security fixes are applied to the latest commit on the default branch.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Use one of the following channels:

1. GitHub Security Advisories (preferred): open a private advisory in this repository.
2. If advisories are unavailable, contact maintainers privately through GitHub.

Please include:

- Affected component/file.
- Reproduction details and impact.
- Any proof-of-concept code or logs.

We will acknowledge reports as quickly as possible, validate severity, and provide remediation guidance.
